I recently integrated Zettelkasten into my 15+ year collection of org notes. Following Jethro Kuan's slip box approach, I organized my notes into four categories:

  1. main - Quick thoughts and snippets
  2. references - Ideas from books, articles, and other sources
  3. article - Long-form content like paper reviews and essays (synthesis of multiple notes)
  4. project - GTD-style task management

Combining Zettelkasten with GTD

The project directory bridges Zettelkasten and GTD workflows. I use ripgrep with custom org-agenda settings to find TODO items in project files and surface them in my agenda.

Why separate projects from other notes?

  • Keeps zettels conceptual and task-free
  • Prevents org-agenda from scanning thousands of note files, which would be unusably slow
  • Forces clear distinction between concepts (zettels) and actionable tasks (projects)

Recently, I started incorporating Zettelkasten into my org notes from over 15 years. Inspired by this slip box setup from Jethro Kuan (the author of org-roam), I decided to follow a similar structure.

  1. main - Thoughts, quick snippets etc.
  2. references - Useful ideas from resources I consume
  3. article - I use this for things like notes, paper reviews, or any long-form content. More generally, its a cumination of different zettes
  4. project - Special directory for incorporating GTD style workflows with org mode

Incorporating GTD Workflows

While the slipbox setup doesn't fall nicely into the more traditional GTD workflows with org mode and agenda, I find it beneificial to keep these TODOs in a special project directory and also consider them notes. With some ripgrepping and custom org-agenda setup, I scrape the files with TODOs in them and add that to the agenda.

Why not let org-agenda run on all the zettes?

I prefer keeping zettels simple and don't want to mix "tasks" in them. Having a dedicated directory forces me think about "concepts" vs tasks clearly. This explodes the number of files org-agenda has to track, making it very inefficient and near unusuable.
